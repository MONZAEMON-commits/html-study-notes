# 06_semantic_html(Webサイトを構成する主要パーツの解説(意味付けタグ))
## ヘッダー（Header）
役割<br>
ページまたはセクションの“最上部”に置く、導入部分。<br>
サイトのタイトルやロゴ、主要メニューなどが置かれる。<br>
用途<br>
- サイトの顔（ロゴ・タイトル）
- ページ全体の共通メニュー
- グローバルナビ
- 検索ボックス
- ログインボタン<br>

など、ユーザーが必ず必要とする導線をまとめる。<br>

**意味（セマンティクス）**<br>
そのページやセクションの「ヘッダー情報」。<br>
サイトの「顔」にあたる。<br>
**HTMLタグ**<br>
```html
<header></header>
```
### headerに入れてよい要素（Allowed）
- ロゴ (`<img>`)
- 見出し (`<h1>〜<h6>`)
- ナビゲーション (`<nav>`)
- テキスト (`<p>`)
- フォーム (`<form>`)
- アイコン（SVG など）
- フッタ以外の一般的なブロック要素
### header入れてはいけない要素と理由（Forbidden）
| NG要素                   | 理由（最重要）                                       |
| :----------------------- | :--------------------------------------------------- |
| `<header>`（ネスト禁止） | header は「導入」。導入を導入で囲むのは意味矛盾      |
| `<footer>`               | header＝導入 / footer＝締め → 役割が逆で構造破壊     |
| `<main>`                 | main＝ページ主役。導入内部に主役を入れるのは階層矛盾 |

**注意点**
- ページに複数置くことは可能（セクションごとに存在して良い）。
- ただしページ全体のトップに置くものは1つだけにするのが一般的。

**HTML構造例**
```html
<header class="site-header">
  <h1 class="logo">My Website</h1>
  <nav class="global-nav">
    <ul>
      <li><a href="/">ホーム</a></li>
      <li><a href="/about">会社概要</a></li>
      <li><a href="/contact">お問い合わせ</a></li>
    </ul>
  </nav>
</header>
```
## ナビゲーション（nav）
役割<br>
サイト内のページ移動を行うためのリンク集。<br>
用途<br>
- 全ページ共通の移動（グローバルナビ）
- 特定カテゴリ内での移動（ローカルナビ）
- 現在位置の提示（パンくずナビ）
- ページ末尾の関連リンク（フッターナビ）
- サイドバーのカテゴリ一覧（ローカルナビ）

など 種類によって役割が違う。<br>
ユーザーを“次のページへ誘導”する領域。<br>
**HTMLタグ**
```html
<nav></nav>
```
**注意点**
- `<nav>` が複数あってもOK<br>
`（ヘッダー／サイドバー／フッターそれぞれに置いて構わない）`<br>

### navに入れてよい要素
- `<ul><li>` のリスト
- `<a>` リンク
- `<button>`（SP時のメニュー開閉など）
- テキスト（補足）
- SVGアイコン

### navに入れてはいけない要素と理由
| NG要素                    | 理由（最重要）                                                     |
| :------------------------ | :----------------------------------------------------------------- |
| `<main>`                  | main＝主役。nav＝導線。導線内部に主役を入れるのは意味不一致        |
| `<article>`（大規模本文） | nav は“移動リンクの集合”。本文を入れると目的が破壊される           |
| 長文の本文テキスト        | ナビ内に本文を書くとセマンティクス崩壊（スクリーンリーダーが混乱） |


**HTML構造例**
```html
<nav class="global-nav">
  <ul>
    <li><a href="/">ホーム</a></li>
    <li><a href="/service">サービス</a></li>
    <li><a href="/news">ニュース</a></li>
  </ul>
</nav>
```
## メインエリア（main）
役割<br>
ページごとに 最も重要な内容を置く場所。<br>
用途<br>
- 記事
- 商品情報
- お知らせ
- 問い合わせ内容

ユーザーがそのページを訪れる主目的の情報を置く。<br>

**HTMLタグ**
```html
<main></main>
```
**注意点**
ページに1回だけ使用可能（HTML仕様上のルール）
### mainに入れてよい要素
- `<article>`（記事）
- `<section>`（区切り）
- `<div>`
- 見出し (`<h2>`〜)
- テキスト (`<p>`)

### mainに入れてはいけない要素と理由

| NG要素                    | 理由（最重要）                                               |
| :------------------------ | :----------------------------------------------------------- |
| `<header>`                | header＝導入。main＝本文。導入を本文の中に入れるのは逆転     |
| `<footer>`                | footer＝締め。本文内部に締めを入れるのは構造矛盾             |
| `<nav>`（グローバルナビ） | main はページ固有の本編、nav はページ全体の導線 → 役割が衝突 |
| `<main>`（ネスト）        | main は1ページ1つのみと仕様に明記                            |

**HTML構造例**
```html
<main>
  <h2>ニュース一覧</h2>
  <p>最新のお知らせを掲載しています。</p>
</main>
```

## 記事エリア（article）
役割<br>
- 独立して成立する情報のまとまり。
- ブログ記事・ニュース記事などが該当。
用途<br>
- ブログ記事
- 掲載ニュース
- コンテンツ（動画・商品案内）
1つの記事が他のページでも意味が通るもの<br>

**HTMLタグ**
```html
<article></article>
```
### articleに入れてよい要素
- 見出し (`<h2>`など)
- `<p>`
- `<img>`
- `<section>`
- `<aside>`（記事補足に限る）

### articleに入れてはいけない要素と理由
| NG要素                      | 理由（最重要）                                                     |
| :-------------------------- | :----------------------------------------------------------------- |
| `<main>`                    | main＝ページ全体の主役、article＝主役内の1コンテンツ → 階層逆転    |
| ページ全体の `<footer>`     | 記事内にページ締めが入ると “記事がページ全体を飲み込む状態” になる |
| 大規模ナビ（グローバルnav） | 記事は本文。グローバルナビはページ構造。意味的に混ざらない         |
**※ 記事内部の“前後ナビ”などのローカルナビはOK**

**HTML構造例**
```html
<article>
  <h2>イベント開催のお知らせ</h2>
  <p>来月より〜</p>
</article>
```
**注意**
単なる段落には使わない<br>
`→ 記事・投稿・レビューなど“独立性”がある内容に使用`

## サイドバー（aside）
役割<br>
メインコンテンツの 補足情報 を表示する領域。<br>
主役ではないが関連性のある情報をまとめる部分。<br>
用途<br>
- 関連記事
- 広告
- プロフィール
- カテゴリ一覧

メインコンテンツの補助的要素<br>
ナビ・広告・プロフィール欄などを置く<br>

**HTMLタグ**
```html
<aside></aside>
```

### asideに入れてよい要素
- リスト (`<ul> / <ol>`)
- 小見出し (`<h3>` など)
- テキスト
- ナビ（ローカルナビとしてはOK）
- 画像

 ### aside入れてはいけない要素と理由
 | NG要素                  | 理由（最重要）                                                |
 | :---------------------- | :------------------------------------------------------------ |
 | `<main>`                | main＝主役。aside＝補助。補助の中に主役は入らない（意味破壊） |
 | `<article>`（記事本体） | aside＝補足。記事本体を入れると役割が逆転する                 |

**HTML構造例**
```html
<aside>
  <h3>カテゴリ</h3>
  <ul>…</ul>
</aside>
```
## フッター（footer）
役割<br>
ページまたはセクションの“締め”を表す区域。<br>
### footerに入れてよい要素
- `<address>`
- ナビ（補助リンク）
- SNSリンク
- ロゴ
- 文章
### footer入れてはいけない要素と理由
| NG要素     | 理由（最重要）                                               |
| :--------- | :----------------------------------------------------------- |
| `<header>` | footer＝締め、header＝導入 → 意味が反対で矛盾                |
| `<main>`   | footerはページやセクションの“終わり”。本文を入れるのは層逆転 |

**HTML構造例**
```html
<footer>
  <p>© 2025 Example</p>
</footer>
```
## 全タグ統一フォーマットOK,NG表
| タグ        | OK要素（入れてよい）                                        | NG要素（入れてはいけない）                   | 理由（最重要）                                                          |
| ----------- | ----------------------------------------------------------- | -------------------------------------------- | ----------------------------------------------------------------------- |
| **header**  | 見出し（h1〜h6）、ロゴ、nav、検索フォーム、p など           | header（ネスト）, footer, main               | 導入(header) の中に導入や締め/footer/本編(main) が入ると構造破壊        |
| **nav**     | ul/li, a, アイコン, ボタン, 小見出し（補足）                | main, article（本文）, 長文テキスト          | nav＝導線 / main＝主役 → 導線の中に本文があると意味衝突                 |
| **main**    | article, section, div, hタグ, p など                        | header, footer, nav（グローバル）, main      | 本編の中に導入(header)と締め(footer)はNG。mainは1つと仕様に明記         |
| **article** | hタグ, p, img, section, aside（補足）, 小ナビ（ローカル）   | main, ページ全体のfooter, グローバルnav      | article＝本文単位。ページ全体の情報を含めると階層逆転                   |
| **aside**   | 補足情報（リスト、広告、プロフィール、ローカルナビ）        | main, article本文                            | aside＝補助、main＝主役 → 主役を補助に入れると意味破壊                  |
| **footer**  | address, nav（補助リンク）, p, ロゴ, SNSリンク              | header, main                                 | footer＝締め、header＝導入 → 真逆。mainを締め内に入れると階層矛盾       |
| **section** | 見出し（必須）, p, img, article, aside（補足）, ローカルnav | ページ全体のheader/footer/main/グローバルnav | section＝ページ内の節。ページ全体の構造要素を入れると意味衝突・階層逆転 |

## ナビゲーションの種類（用途によって呼び方が変わる）
### グローバルナビゲーション（Global Navigation）
用途<br>
- サイト全体を移動するメインナビ
- どのページにも共通して表示される<br>
例<br>
「ホーム｜会社概要｜サービス｜採用｜お問い合わせ」<br>
**HTML構成**<br>
通常は `<header>` 内の `<nav>` に置く。
```html
<header>
  <nav class="global-nav">
    <ul>
      <li><a href="#">ホーム</a></li>
      <li><a href="#">サービス</a></li>
      <li><a href="#">会社概要</a></li>
      <li><a href="#">お問い合わせ</a></li>
    </ul>
  </nav>
</header>
```
### ローカルナビゲーション（Local Navigation）
用途<br>
- そのカテゴリやページ内限定のメニュー<br>
例：<br>
「サービス」ページの内部リンク一覧<br>
例:<br>
サービス一覧 →「Web制作」「アプリ開発」「AIソリューション」<br>
**HTML構成**<br>
サイドバーに置くのが一般的。<br>
```html
<aside class="local-nav">
  <nav>
    <ul>
      <li><a href="#">Web制作</a></li>
      <li><a href="#">アプリ開発</a></li>
      <li><a href="#">AIソリューション</a></li>
    </ul>
  </nav>
</aside>
```
### パンくずナビゲーション（Breadcrumb Navigation）
用途<br>
- ユーザーがサイト内の“どこにいるか”を示すナビ<br>
- パンくずリスト（Breadcrumbs）とも呼ばれる<br>
- 階層が深いサイトで必須<br>

表示例<br>
ホーム > サービス > Web制作 > 料金表<br>
**HTML構成**
nav + ol（順序リスト）を使うとセマンティック。
```html
<nav aria-label="パンくずリスト">
  <ol class="breadcrumb">
    <li><a href="#">ホーム</a></li>
    <li><a href="#">サービス</a></li>
    <li>Web制作</li>
  </ol>
</nav>
```
## <section>（セクション）＋ アウトラインの解説
役割<br>
`Webページを意味のあるまとまり（章・節）ごとに区切るためのタグ。`<br>
ページを「見た目」ではなく「意味（セマンティクス）」で整理するために使う。<br>
用途<br>
- サービス紹介セクション
- 料金セクション
- お問い合わせセクション
- “見出しを付けられる意味のまとまり” ならなんでも
- トップページで複数ブロックを区切るときに使う<br>

`<section>` は「文書の一部を“節（セクション）”として定義する」ための要素。<br>
“意味のあるまとまり”に名前（見出し）がつけられるものに使う。<br>
例：「特徴」「このサイトについて」「お問い合わせ」<br>

## **アウトラインとの関係（最重要）**
`<section>` は 文書のアウトライン（構造）を形成するためのタグ<br>
アウトラインとは：<br>
見出し（h1〜h6）を基準に文書全体を階層構造として整理したもの。<br>
検索エンジン・スクリーンリーダーが ページ内容を理解するための論理構造。<br>

## `<section>` がアウトラインに与える影響
- `<section>` の開始＝アウトラインに「節（Chapter）」が追加される
- セクション内にある見出し（hタグ）が、そのままアウトラインのタイトルになる
- ページ全体を論理的なブロックに分けられる<br>

見出し（hタグ）は必須？<br>
`実質必須（W3C(世界ルール的なもの)も推奨）`<br>
理由：<br>
`<section>` は “意味のあるまとまり” を作るので、<br>
そのまとまりの タイトルが必要（= 見出し）。<br>

例：
```html
<section>
  <h2>サービス紹介</h2>  ← この見出しがアウトラインの節タイトルになる
  <p>説明…</p>
</section>
```
アウトライン例：
```html
<h1>会社案内</h1>

<section>
  <h2>事業内容</h2>
  <p>…</p>
</section>

<section>
  <h2>アクセス</h2>
  <p>…</p>
</section>
 <!-- <section> は文書構造に直接影響する。 -->
```
### sectionに入れてよい要素
`<section>` は “意味のあるまとまり” なので：
- `<h1>〜<h6>`（※ほぼ必須）
- `<p>`
- `<img>`
- `<article>`
- `<div>`
- `<ul>, <ol>`
- `<aside>`（補足としてならOK）
- `<nav>`（セクション内部専用のローカルナビならOK）
**※ つまり一般的なフローコンテンツはほぼ何でも入れられる。**
### sectionに入れてはいけない要素と理由
| NG要素                                     | 理由（最重要）                                                 |
| :----------------------------------------- | :------------------------------------------------------------- |
| **ページ全体の `<header>`**                | header＝導入、section＝意味のまとまり → 役割が違うため不適切   |
| **ページ全体の `<footer>`**                | section の内部に “ページ締め” を入れるのは構造矛盾             |
| **`<main>`**                               | main はページ全体の主役 → section はその下位概念なので階層逆転 |
| **ページ全体の `<nav>`（グローバルナビ）** | nav は全体構造の案内、section 内にはローカルナビのみ許容       |

**注意点**
- `<section>` は乱用禁止<br>
→ 意味のあるまとまりにだけ使う<br>
（ただの箱なら `<div>` を使う）<br>

- 見出し（hタグ）は 必ず置く<br>
→ 置かないとアウトラインが正常に形成されない<br>

- `<section>` と `<article>` の違い<br>
article＝独立したコンテンツ<br>
section＝ページ内の意味ブロック<br>

## `<time>`（日時を表すセマンティックタグ）
役割<br>
日時情報を、機械にも人にも正しく伝えるためのタグ。<br>
用途<br>
- 記事の日付
- イベント日時
- 更新日
- 公開日の構造化データとして検索エンジンに渡す<br>

 `ブラウザ・検索エンジン・スクリーンリーダーに「これは日時です」 と明示する。`<br>

 | タグ | OK要素（入れてよい）               | NG要素（入れてはいけない）                | 理由（最重要）                                                                            |
 | ---- | ---------------------------------- | ----------------------------------------- | ----------------------------------------------------------------------------------------- |
 | time | 日付テキスト、数字、インライン要素 | 画像（img）、div/section などブロック要素 | `<time>` は“日時”の意味付けをするタグ。日時以外の内容を入れるとセマンティクスが破壊される |

**HTML構成**
```html
 <p>公開日：<time datetime="2025-11-10">2025年11月10日</time></p>
```

## `<figure>`（画像・図・表・コード例の“独立ブロック”）
役割<br>
画像やグラフなどを 独立した図版としてグループ化するコンテナ。<br>
用途<br>
- 画像＋説明文（キャプション）
- 図／イラスト
- コード例
- 表（table）を図として使う場合<br>

そのブロックが “独立した図版” であることを示す。<br>
body の本文に依存しない 自立したメディア要素 になる。<br>

| タグ   | OK要素（入れてよい）                  | NG要素（入れてはいけない）          | 理由（最重要）                                                   |
| ------ | ------------------------------------- | ----------------------------------- | ---------------------------------------------------------------- |
| figure | img、figcaption、pre/code、table、svg | header、footer、main、グローバルnav | figureは本文の図版。ページ全体の構造要素を入れると階層矛盾になる |

**HTML構成**
```html
<figure>
  <img src="cat.jpg" alt="猫">
  <figcaption>かわいい猫</figcaption>
</figure>
```
## `<figcaption>`図版（figure）の説明文・タイトル
役割<br>
図版（figure）の説明文・タイトル<br>
用途<br>
- 画像の説明
- 図表の注釈
- 参考情報<br>

`「この図に対する説明・名前」です、と明示する。`

| タグ       | OK要素（入れてよい）    | NG要素（入れてはいけない）                      | 理由（最重要）                                                                   |
| ---------- | ----------------------- | ----------------------------------------------- | -------------------------------------------------------------------------------- |
| figcaption | テキスト、a、strong、em | header、footer、main、divなど大規模ブロック要素 | figcaptionは図の説明専用。構造タグや大きなブロックを入れると説明として成立しない |

**HTML構成**
```html
<figure>
  <img src="chart.png" alt="売上グラフ">
  <figcaption>2025年度売上推移</figcaption>
</figure>
```

## `<small>`重要度の低いテキスト
役割<br>
注釈・補足情報・法的文言など “重要度の低いテキスト” を表す。<br>
用途<br>
- 注意文
- 免責事項
- 備考
- 著作権表示<br>

「これは本内容ではなく補足情報ですよ」という意味付け。

| タグ  | OK要素（入れてよい）                      | NG要素（入れてはいけない）              | 理由（最重要）                                                                |
| ----- | ----------------------------------------- | --------------------------------------- | ----------------------------------------------------------------------------- |
| small | テキスト、インライン要素（a、em、strong） | div、article、sectionなどのブロック要素 | smallはインライン要素。補足情報であり、ブロックを入れると文法・意味が破綻する |

**HTML構成**
```html
<p>このサービスは無料です。<small>※一部機能を除く</small></p>
```
