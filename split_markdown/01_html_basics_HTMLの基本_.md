# 01_html_basics(HTMLの基本)
- 基本の書き方<br>
    <タグ名>～</タグ名>
- 属性と属性値について<br>
<タグ名 付加情報=”内容”>～</タグ名>

- HTMLの骨格の作り方(お決まりセット)<br>
    VSC上で「!」を入力してEnterで以下に変換される。

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    
</body>
</html>
```
**注意事項**
- 本文以外はすべて半角英数字
- タグの互い違いに注意

-お決まりセットの構造
| タグ                                                                     | 役割                             | 詳細な説明                                                                                                                                                                                                                                         |
| ------------------------------------------------------------------------ | -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `<!DOCTYPE html>`                                                        | HTML文書のバージョン宣言         | ブラウザに「このページはHTML5で書かれています」と知らせるための宣言。古いHTML(例：HTML4やXHTML)では別の宣言が必要だったが、HTML5ではこれだけでOK。これがないとブラウザが「互換モード」で表示してしまい、意図しないデザイン崩れを起こす場合がある。 |
| `<html lang="en">`                                                       | HTML全体のルート要素             | ページ全体を囲む最上位のタグ。すべての要素はこの中に含まれる。`lang`属性はページの言語を指定し、音声読み上げソフトや検索エンジンが正しく処理するために使われる。日本語サイトなら `lang="ja"` にするのが一般的。                                    |
| `<head>`                                                                 | ページの設定情報を記述する部分   | ページの表示内容ではなく、**ブラウザや検索エンジンに伝える情報**をまとめる領域。文字コード、タイトル、外部CSSやJavaScriptの読み込み、SEO情報、SNS連携用のメタ情報(OGPなど)をここに記述する。                                                       |
| `<meta charset="UTF-8">`                                                 | 文字コードの指定                 | ページで使う文字のエンコーディングを「UTF-8(ユニコード)」に指定。UTF-8は世界中の文字を扱えるため、国際標準的に推奨される。これがないと文字化けが起こる可能性がある。                                                                               |
| `<meta name="viewport" content="width=device-width, initial-scale=1.0">` | スマホ・タブレット向けの表示設定 | 表示領域(viewport)をデバイス幅に合わせる設定。`width=device-width` でスマホの画面幅に合わせ、`initial-scale=1.0` で初期の拡大率を1倍にする。これがないと、スマホでPCサイズのページが縮小されて表示されてしまう。レスポンシブ対応サイトでは必須。   |
| `<title>Document</title>`                                                | ページタイトル                   | ブラウザのタブ名や検索結果、ブックマークに表示されるページ名。SEO(検索エンジン最適化)的にも非常に重要で、ページ内容を端的に表すタイトルを設定するのが基本。例：`<title>社員体調管理システム｜ポートフォリオ</title>`                               |
| `<body>`                                                                 | 実際に表示されるコンテンツ       | ページ上にユーザーが目にする要素をすべて記述する部分。見出し、段落、画像、リンク、ボタン、フォームなどすべてのUI要素はこの中に書く。CSSやJavaScriptを使ってデザインや動きを加えるのもここで行う。                                                  |
| `</html>`                                                                | HTML文書の終端                   | ページ全体の終了を明示するタグ。`<html>`の閉じタグであり、HTML構造を正しく完結させる役割を持つ。閉じ忘れるとHTML解析エラーが発生することがある。                                                                                                   |

## マークアップについて
`マークアップとは文章や情報に “意味” や “役割” をタグで与えて構造化すること。`
## 基本のタグ

|    タグ名     |                                    解説                                    |
| :-----------: | :------------------------------------------------------------------------: |
| `<h1>～</h1>` | *見出しを表すタグ　hはheading(ヘディング)の頭文字h1～h6までの段階がある。* |
|  `<p>～</p>`  |             *段落を表すタグ　pはparagraph(パラグラフ)の頭文字*             |
|    `<br>`     |                         *改行 break(ブレイク)の略*                         |

## 3種のリストを使い分ける

|                       タグ名                        |                                                      解説                                                       |                            使用例                             |
| :-------------------------------------------------: | :-------------------------------------------------------------------------------------------------------------: | :-----------------------------------------------------------: |
|         `<ul>`<br>`<li>～</li>`<br>`</ul>`          |  *項目の順序に意味を持たないリストを表すタグ(「・」で箇条書き)  ulはunordered list(アンオーダードリスト)の略*   |         <ul><li>箇条書き1</li><li>箇条書き2</li></ul>         |
|         `<ol>`<br>`<li>～</li>`<br>`</ol>`          | *道順・レシピ・順位など順番を固定したいリストを表すタグ(1～nで箇条書き) olはordered list(オーダードリスト)の略* |               <ol><li>A賞</li><li>B賞</li></ol>               |
| `<dl>`<br>`<dt>～</dt>`<br>`<dd>～</dd>`<br>`</dl>` |                 *項目名とそのリストがセットのタグ dlはdescription list(ディスクリプション)の略*                 | <dl><dt>年齢</dt><dd>xx歳</dd><dt>名前</dt><dd>kuma</dd></dl> |

## リンク部のマークアップ

|       タグ名       |                                      解説                                       |           使用例            |
| :----------------: | :-----------------------------------------------------------------------------: | :-------------------------: |
| `<a href="#"><a> ` | href属性でリンク先を指定するタグ(#の部分にリンクを記載) aはanchor(アンカー)の略 | <a href="#">一番上に戻る<a> |

## href 属性の主な指定パターン

| 種類                         | 記述例                                              | 説明                                                          |
| ---------------------------- | --------------------------------------------------- | ------------------------------------------------------------- |
| **絶対URL**                  | `<a href="https://example.com/">サイトへ</a>`       | 完全なURLを指定。外部サイトへリンクするときに使用。           |
| **相対URL(同一サイト内)**    | `<a href="about.html">会社概要</a>`                 | 現在のページを基準にしたパス。サイト内リンクで使う。          |
| **ルート相対パス**           | `<a href="/images/pic.png">画像</a>`                | ドメイン直下を基準に指定。サイト構造が深くても安定。          |
| **ページ内リンク(アンカー)** | `<a href="#section1">詳しく見る</a>`                | 同じページ内のid属性(`id="section1"`)へジャンプ。             |
| **メールリンク**             | `<a href="mailto:info@example.com">メール送信</a>`  | メールソフトを開いて送信先を指定。                            |
| **電話リンク**               | `<a href="tel:09012345678">電話する</a>`            | スマホでタップしたときに電話発信する。                        |
| **JavaScriptリンク(非推奨)** | `<a href="javascript:void(0)">クリック</a>`         | クリックイベント用。ただし現在は`onclick`などを使うのが主流。 |
| **ファイルダウンロード**     | `<a href="data/file.zip" download>ダウンロード</a>` | `download`属性を追加で使うと、直接DLできる。                  |
| **空リンク(仮置き)**         | `<a href="#">リンク</a>`                            | 仮のリンク。クリックでページ上部に戻る。開発中に一時利用。    |

## HTMLでよく使う画像形式まとめ

| 形式     | 主な拡張子      | 特徴・用途                                                                   | 透明対応 | アニメ対応    |
| -------- | --------------- | ---------------------------------------------------------------------------- | -------- | ------------- |
| **JPEG** | `.jpg`, `.jpeg` | 写真やグラデーションに最適。圧縮率が高く軽い。非可逆圧縮なので画質劣化あり。 | ❌        | ❌             |
| **PNG**  | `.png`          | 背景透過やロゴ・アイコンに最適。可逆圧縮で劣化なし。ファイルはやや重い。     | ✅        | ❌             |
| **GIF**  | `.gif`          | アニメーション対応。色数は256色まで。軽量な短い動きに向く。                  | ✅        | ✅             |
| **SVG**  | `.svg`          | ベクター画像。拡大縮小しても劣化なし。ロゴ・アイコン・図形に最適。           | ✅        | ✅(CSS/JSで可) |
| **WebP** | `.webp`         | Google開発の新形式。JPEGより軽く、透過・アニメ両対応。モダンブラウザ対応。   | ✅        | ✅             |
| **AVIF** | `.avif`         | 次世代形式。高圧縮で高画質。WebPより軽いが非対応ブラウザも残る。             | ✅        | ✅             |
| **ICO**  | `.ico`          | ファビコン(ブラウザタブのアイコン)専用。16×16などの小サイズ用。              | ✅        | ❌             |

## 強調したい文章

|                                 タグ名                                 |                                                                                                        解説                                                                                                        |                                       使用例                                        |
| :--------------------------------------------------------------------: | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------: | :---------------------------------------------------------------------------------: |
|                             `<em>～</em>`                              | *文章に協調のニュアンスを与えるタグ*emは “emphasis(エンファシス)” の略で、「強調する」という意味。視覚的には斜体で表示されるが、意味的には「感情的な強調」を示す。スクリーンリーダーでも強調として読み上げられる。 |                     <p>これは<em>強調された</em>部分です。</p>                      |
|                         `<strong>～</strong>`                          |                                                                                       *より強い強調を表すタグ(重要性を示す)*                                                                                       |                   <p>これは<strong>非常に重要</strong>です。</p>                    |
|                           `<mark>～</mark>`                            |                                                                                *文字にハイライト(蛍光ペン)のような強調を与えるタグ*                                                                                |                    <p>これは<mark>注目</mark>すべき点です。</p>                     |
|                              `<i>～</i>`                               |                                                                               *イタリック(斜体)を表すタグ。感情や専門用語などに使う*                                                                               |                      <p>英語では<i>Italic</i>と書きます。</p>                       |
|                              `<b>～</b>`                               |                                                                                   *ボールド(太字)を表すタグ。視覚的な強調に使用*                                                                                   |                           <p>これは<b>太字</b>です。</p>                            |
| `<table>～</table>`<br>`<tr>～</tr>`<br>`<th>～</th>`<br>`<td>～</td>` |        *表(テーブル)を作成するタグ。tableは「表」、trは「行(table row)」、thは「見出しセル(table header)」、tdは「データセル(table data)」を意味する。構造的なデータや一覧を整理して表示するのに使われる。*        | <table><tr><th>名前</th><th>年齢</th></tr><tr><td>kuma</td><td>30</td></tr></table> |
|                        `<address>～</address>`                         |           *連絡先や著者情報を記載するためのタグ。addressは「住所」「連絡先情報」を意味し、一般的には斜体で表示される。メールアドレス、URL、住所、署名などに使用され、文書や記事の末尾などに配置される。*           |        <address>福岡市博多区〇〇町<br>メール：example@example.com</address>         |
